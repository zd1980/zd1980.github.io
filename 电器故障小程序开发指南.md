# 电器故障查询微信小程序开发指南

## 1. 项目目标
你的小程序核心流程是：
1. 用户输入：电器大类、子类、品牌、故障代码。
2. 小程序调用后端接口查询数据库。
3. 返回故障说明和排查建议。
4. 管理员在后台可新增/修改/删除故障信息。

---

## 2. 推荐技术方案（适配你当前环境）
你已有 Win10 + 微信开发者工具 + VS Code + 小程序 AppID，推荐用 **微信云开发（CloudBase）**，因为：
- 不需要自己搭服务器和域名备案（前期成本最低）。
- 数据库、云函数、鉴权一体化，适合个人/小团队快速上线。
- 后续可平滑迁移到自建后端（Java/Python/Node）架构。

### 技术栈建议
- 前端：微信小程序原生（WXML + WXSS + JS）
- 后端：云函数（Node.js）
- 数据库：云开发数据库（NoSQL）
- 管理端：
  - 方案A（推荐起步）：小程序内做“管理员页面”
  - 方案B（后期）：独立 Web 管理后台（Vue/React）

---

## 3. 功能拆解

## 3.1 用户端（前端）
页面字段：
- 电器大类（例如：空调、冰箱、洗衣机）
- 电器子类（例如：挂机、柜机）
- 电器品牌（例如：格力、美的）
- 故障代码（例如：E6、F1）

按钮：
- `立即查询`

查询结果区：
- 标准故障码（如 E6）
- 故障含义
- 排查步骤（1、2、3）
- 备注（是否通用/仅某型号）

## 3.2 管理端（后台）
管理员可操作：
- 新增故障记录
- 编辑故障记录
- 删除故障记录
- 按品牌/代码搜索记录
- 批量导入（后期可做 Excel 导入）

权限控制：
- 只有管理员 openid 可以访问管理页和管理接口。

## 4. 数据库设计（核心）
建议集合名：`fault_codes`

单条记录结构示例：
```json
{
  "category": "空调",
  "subCategory": "挂机",
  "brand": "格力",
  "faultCode": "E6",
  "faultMeaning": "通讯故障",
  "solutionSteps": [
    "先检查内外机连接线是否短路",
    "如连接线正常，再检查外机变频板"
  ],
  "modelScope": "部分机型",
  "createdAt": "2026-01-01T10:00:00.000Z",
  "updatedAt": "2026-01-01T10:00:00.000Z",
  "createdBy": "管理员openid"
}
```

查询组合索引建议（提升速度）：
- `category + subCategory + brand + faultCode`

注意规范：
- `faultCode` 建议统一转大写（E6、D6），避免大小写导致查不到。
- 增加字段 `aliases`（可选），用于兼容历史故障码写法。

## 5. 接口设计（云函数）

## 5.1 用户查询接口
云函数名：`queryFault`

入参：
```json
{
  "category": "空调",
  "subCategory": "挂机",
  "brand": "格力",
  "faultCode": "E6"
}
```

返回：
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "faultMeaning": "通讯故障",
    "solutionSteps": [
      "先检查内外机连接线是否短路",
      "如连接线正常，再检查外机变频板"
    ]
  }
}
```

未查询到：
```json
{
  "code": 404,
  "message": "未找到该故障码，请核对输入信息"
}
```

## 5.2 管理接口
- `addFault`：新增
- `updateFault`：修改
- `deleteFault`：删除
- `listFaults`：分页查询

每个管理接口都要先校验管理员身份（openid 白名单）。

---

## 6. 前端页面结构建议
建议至少 3 个页面：
1. `pages/query/index`：用户查询页
2. `pages/result/index`：结果页（也可放在查询页下方）
3. `pages/admin/index`：管理员管理页

用户查询页交互要点：
- 表单校验：四个字段不能为空
- 查询按钮防重复点击（loading 状态）
- 失败提示：Toast + 明确错误原因
- 输入自动纠错：故障码自动大写

### 7.2 云函数查询逻辑（Node.js 伪代码）
```javascript
const db = cloud.database()
const _ = db.command

const result = await db.collection('fault_codes').where({
  category,
  subCategory,
  brand,
  faultCode: faultCode.toUpperCase().trim()
}).limit(1).get()

if (!result.data.length) {
  return { code: 404, message: '未找到该故障码，请核对输入信息' }
}

return { code: 0, message: 'ok', data: result.data[0] }
```

## 8. 上线前必须做的 8 项检查
1. 故障码大小写兼容（E6/e6）
2. 空格处理（用户输入前后空格）
3. 高并发下查询性能（至少压测 100 并发）
4. 管理员权限校验是否可绕过
5. 数据备份策略（每天自动导出）
6. 错误日志记录（云函数日志）
7. 敏感词/违规内容审核
8. 用户隐私合规文档（隐私政策）
