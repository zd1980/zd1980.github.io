电器故障查询微信小程序：从 0 到上线

目标：用户输入「电器大类 / 子类 / 品牌 / 故障代码」，小程序查询数据库并返回故障信息。

推荐技术栈：微信小程序 + Node.js + MySQL 接口风格：RESTful 部署建议：腾讯云 CloudBase / CVM

1) 功能拆解与开发顺序

步骤 A：需求建模
梳理字段：category、subCategory、brand、faultCode、faultDesc、cause、solution、safetyTip。
步骤 B：数据库设计
先做可查询、可扩展的数据表；加组合索引保障查询速度。
步骤 C：后端 API
实现 /api/faults/query，做参数校验、模糊匹配、错误码规范。
步骤 D：小程序页面
输入表单 + 查询按钮 + 结果卡片 + 空状态 + 错误提示。

2) 推荐数据库结构（MySQL）

CREATE TABLE appliance_faults (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  category VARCHAR(50) NOT NULL COMMENT '电器大类，如空调',
  sub_category VARCHAR(50) NOT NULL COMMENT '电器子类，如挂机',
  brand VARCHAR(50) NOT NULL COMMENT '品牌，如格力',
  fault_code VARCHAR(50) NOT NULL COMMENT '故障代码，如E1',
  fault_desc VARCHAR(255) NOT NULL COMMENT '故障说明',
  possible_cause TEXT COMMENT '可能原因',
  solution TEXT COMMENT '处理建议',
  safety_tip VARCHAR(255) COMMENT '安全提示',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_fault (category, sub_category, brand, fault_code),
  KEY idx_lookup (category, sub_category, brand, fault_code)
);

如果后期要支持“同代码多机型”，可把机型 model 单独扩展为新字段或关联表。

3) 后端查询接口示例（Node.js/Express）

app.post('/api/faults/query', async (req, res) => {
  const { category, subCategory, brand, faultCode } = req.body;
  if (!category || !subCategory || !brand || !faultCode) {
    return res.status(400).json({ code: 40001, message: '参数不完整' });
  }

  const [rows] = await db.execute(
    `SELECT category, sub_category, brand, fault_code,
            fault_desc, possible_cause, solution, safety_tip
       FROM appliance_faults
      WHERE category = ? AND sub_category = ?
        AND brand = ? AND fault_code = ?
      LIMIT 1`,
    [category.trim(), subCategory.trim(), brand.trim(), faultCode.trim().toUpperCase()]
  );

  if (rows.length === 0) {
    return res.status(404).json({ code: 40401, message: '未找到该故障代码' });
  }

  return res.json({ code: 0, data: rows[0] });
});
